# ImiTrade Frontend — Архитектура и правила

## Стек

- React 19
- TypeScript
- Vite
- React Router
- Axios
- TanStack Query
- Material UI (MUI)
- React Hook Form + Zod
- Sonner
- Lucide React

## Архитектура

Используется **feature-based architecture**.

```
src/
├── app/           # Инициализация приложения: Router, глобальные провайдеры, App.tsx
├── pages/         # Страницы, привязанные к маршрутам
├── widgets/       # Самостоятельные блоки UI, составленные из фич и shared
├── features/      # Изолированные бизнес-сущности (auth, portfolio, trading, …)
│   └── <feature>/
│       ├── api/        # API-запросы фичи
│       ├── components/ # Компоненты, специфичные для фичи
│       ├── hooks/      # Хуки, специфичные для фичи
│       ├── types/      # Типы, специфичные для фичи
│       └── lib/        # Утилиты и валидаторы фичи
└── shared/        # Переиспользуемый код, не привязанный к бизнес-логике
    ├── api/         # Общий ApiClient и базовые настройки HTTP
    ├── components/  # Переиспользуемые UI-компоненты
    ├── hooks/       # Общие хуки
    ├── lib/         # Общие утилиты (queryClient, helpers)
    ├── providers/   # Глобальные React-провайдеры (Theme, Query, Toast)
    ├── styles/      # Темы, глобальные стили, CSS-константы
    ├── types/       # Глобальные TypeScript-типы
    └── utils/       # Чистые функции-утилиты
```

## Правила размещения компонентов

1. **Если компонент используется только внутри одной фичи** — он живёт в `features/<feature>/components/`.
2. **Если компонент используется в двух и более фичах** — он поднимается в `shared/components/`.
3. **Страницы** живут в `pages/` и не содержат бизнес-логики — только компоновку виджетов.
4. **Widgets** — сложные UI-блоки (шапка, сайдбар, карточка портфеля), которые могут объединять несколько фич.

## Где хранится API

- `shared/api/` — общий `ApiClient` (Axios), базовые типы ответов, интерсепторы (JWT, Guest Token и т.д.).
- `features/<feature>/api/` — эндпоинты, хуки `useQuery`/`useMutation`, типы запросов/ответов конкретной фичи.

## Где хранится общая логика

- `shared/hooks/` — общие хуки (useDebounce, useLocalStorage и т.д.).
- `shared/lib/` — `queryClient`, вспомогательные функции.
- `shared/utils/` — чистые функции (форматтеры, валидаторы).

## Где хранить типы

- `shared/types/` — глобальные типы, интерфейсы, используемые в нескольких фичах.
- `features/<feature>/types/` — типы, которые принадлежат только одной фиче.
- **Запрещено** импортировать типы из фичи A в фичу B. Если тип нужен в нескольких фичах — он поднимается в `shared/types/`.

## Запрещено использовать

- Redux
- MobX
- Zustand
- Bootstrap
- Любые другие глобальные state-менеджеры вне React Context + TanStack Query

## Правило: пишем код небольшими feature

- Каждая фича — это закрытый модуль.
- Фича не должна импортировать внутренние файлы другой фичи.
- Общение между фичами происходит только через `shared/` (типы, утилиты, события).
- При росте фичи дробим её на подфичи или выносим код в `widgets/` / `shared/`.

---

*Без описания бизнес-логики. Для разработки отдельных фич смотри соответствующие модули в `features/`.*
