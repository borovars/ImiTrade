## План: Онбординг (WelcomeDialog) + страница About

Реализация полностью на frontend, без изменения backend и бизнес-логики. Шаги выполняются в порядке зависимостей.

---

### 1. Регистрация фирменного цвета в теме MUI
**Файл:** `frontend/src/shared/styles/theme.ts` (изменить)

Сейчас тема содержит только `palette.mode: 'light'` (AppBar/кнопки — дефолтный синий). Добавлю `palette.primary.main = '#3dba8d'` (+ `dark`/`light`/`contrastText` для корректных hover-состояний). Следствие: AppBar топбара, кнопки `color="primary"`, чипы и т.д. станут фирменного зелёного цвета во всём приложении — осознанное изменение по решению пользователя. `#3dba8d` уже используется в графиках, так что палитра станет консистентной.

### 2. Расширение централизованной утилиты storage
**Файл:** `frontend/src/shared/lib/storage.ts` (изменить)

Добавлю ключ `ONBOARDING_COMPLETED: 'imitrade_onboarding_completed'` в `STORAGE_KEYS` и методы `isOnboardingCompleted(): boolean` / `setOnboardingCompleted(): void` в объект `storage`. Прямые вызовы `localStorage` в компонентах не допускаются (конвенция проекта — всё через `storage.ts`).

### 3. Компонент онбординг-диалога
**Файл:** `frontend/src/features/onboarding/ui/WelcomeDialog.tsx` (создать)

Полностью независимый компонент (не смешивается с Dashboard/страницами). Логика:
- `useState(() => !storage.isOnboardingCompleted())` — синхронно читает флаг при первом рендере (SPA без SSR, флэша не будет). Если онбординг не пройден — `open=true`.
- MUI `Dialog` (`maxWidth="xs" fullWidth` — как `TradeStockDialog`).
- **Заголовок-вардрмарк**: текст «ImiTrade» стилизованный фирменным цветом (первичный теперь зелёный) + подзаголовок «Добро пожаловать в ImiTrade». Текстовый логотип, без ассета (по решению пользователя).
- **Описание** (из ТЗ): про учебный симулятор, виртуальные кредиты, MOEX, покупку/продажу, портфель.
- **Блок стартового капитала**: выделенная плашка (`Box bgcolor: 'action.selected'`) с двумя буллетами — 5 000 при первом посещении, +20 000 после регистрации.
- **Кнопка «Начать»** (`Button variant="contained" color="primary"` → фирменный зелёный) — вызывает `storage.setOnboardingCompleted()` и закрывает диалог.
- **Ссылка «Подробнее о проекте»** внизу — `useNavigate('/about')`, предварительно сохраняет флаг и закрывает диалог.
- Цвета завязаны на токены темы (`text.secondary`, `action.selected`, `primary.main`), без хардкод-hex — соответствует единым стандартам UX.

### 4. Монтирование WelcomeDialog
**Файл:** `frontend/src/widgets/layout/AppLayout.tsx` (изменить)

Добавлю `<WelcomeDialog />` рядом с `<AppTopBar />`. AppLayout оборачивает layout-группу маршрутов (`/dashboard`, `/stocks`, `/portfolio`, `/transactions`, `/about`). При первом визите редирект `/` → `/dashboard` попадает в layout, поэтому диалог покажется. Компонент сам управляет своей видимостью через storage, props не требует.

### 5. Страница About
**Файл:** `frontend/src/pages/AboutPage.tsx` (создать)

MUI, адаптивная вёрстка, контент разбит на отдельные `Card` (не длинные текстовые блоки). Разделы из ТЗ:
1. **Что такое ImiTrade** — краткое описание.
2. **Возможности** — список (покупка/продажа, виртуальный счёт, портфель, история, инфо о компаниях, графики, гостевой режим, регистрация).
3. **Откуда берутся данные** — MOEX ISS API, виртуальные деньги.
4. **Используемые технологии** — три подкатегории (Backend / Frontend / Интеграции) списками.
5. **Архитектура** — feature-based на backend и frontend.
6. **Назначение проекта** — pet-проект для демонстрации навыков.

Лейаут: `Container maxWidth="lg"` + отзывчивая сетка карточек (через MUI `Grid` нового API или `Box` с CSS-grid — уточню по существующим паттернам в кодовой базе при реализации, чтобы совпасть с принятым в проекте способом). Заголовок-вардрмарк «ImiTrade» стилизованный фирменным цветом в шапке страницы. Кнопка «Назад» не нужна — топбар всегда виден.

### 6. Маршрут /about
**Файл:** `frontend/src/app/AppRouter.tsx` (изменить)

Добавить `<Route path="/about" element={<AboutPage />} />` **внутрь** layout-группы (`<Route element={<AppLayout />}>`), рядом с `/dashboard` и т.д. — чтобы у страницы был топбар и сработал онбординг (если вдруг пользователь попал на /about первым).

### 7. Ссылка в Header
**Файл:** `frontend/src/widgets/topbar/AppTopBar.tsx` (изменить)

Добавить в `NAV_ITEMS` запись `{ label: 'О проекте', path: '/about', icon: <Info size={20} /> }` (импортировать `Info` из `lucide-react`). Получит `NavLink`, иконку и `.active`-подсветку как остальные пункты. `end` не нужен (маршрут точный).
