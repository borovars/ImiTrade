## План: Адаптивный Header с бургер-меню и меню пользователя

Рефакторинг только UI (Header); маршруты, auth-логика и бизнес-логика не меняются. Логика `useLogout` переиспользуется как есть.

---

### Текущее состояние
`widgets/topbar/AppTopBar.tsx` (~190 строк) — один файл: бренд + кнопка About (слева), навигация по центру (абсолютное центрирование, 4 пункта), блок справа (Активы + Chip + Login/Register или username+Выйти). На узких экранах центральная навигация и правый блок конфликтуют.

---

### Новая структура файлов
Разделю на подкомпоненты внутри `widgets/topbar/` (держу близко к текущему расположению, а не отдельную папку `Header/` — меньше хаоса, совпадает с feature-based структурой проекта):

```
widgets/topbar/
├── AppTopBar.tsx        (изменить — композитор)
├── navItems.ts          (создать — общий массив NAV_ITEMS для desktop и drawer)
├── DesktopNav.tsx       (создать — центральная навигация md+)
├── MobileNav.tsx        (создать — бургер + Drawer)
└── UserMenu.tsx         (создать — меню аккаунта/гостя)
```

### 1. `navItems.ts` (создать)
Вынесу существующий массив `NAV_ITEMS` + тип `NavItem` в отдельный модуль, чтобы DesktopNav и MobileNav (Drawer) делили один источник и не было расхождений. About в массив НЕ входит — он остаётся иконкой у бренда (по решению прошлого шага). Иконки — lucide-react (`LayoutDashboard, TrendingUp, Wallet, Receipt`).

### 2. `DesktopNav.tsx` (создать)
Центральная навигация для `md+`. Копия текущего блока (абсолютное центрирование через `position:'absolute', left:'50%'`). Оборачиваю в скрытие `{ display: { xs: 'none', md: 'flex' } }` — на mobile отсутствует.

### 3. `MobileNav.tsx` (создать)
Бургер + Drawer для `xs/sm`:
- `IconButton` (бургер `Menu` из lucide) с `{ display: { xs: 'flex', md: 'none' } }`.
- Состояние `drawerOpen` локально (`useState`).
- MUI `Drawer` (`anchor="left"`), внутри — `List` пунктов из `navItems`. Каждый пункт — `ListItemButton component={NavLink}`; клик закрывает Drawer (`onClose`).
- Активный пункт подсвечивается `.active` NavLink (консистентно с desktop).
- About в Drawer НЕ дублирую (он уже иконка в шапке у бренда, доступен на любом размере).

### 4. `UserMenu.tsx` (создать)
Единое меню пользователя (замена разрозненным кнопкам). Только UI, без auth-логики — получает колбэк `onLogout: () => void` и читает `useAuth().state`.
- Триггер: MUI `Button` с `Avatar`-иконкой пользователя (`User` из lucide в круге) + подпись + стрелка `ChevronDown`.
  - **Guest**: подпись «Гость» → меню с пунктами «Регистрация (+20 000)» → `/register`, «Вход» → `/login`.
  - **Auth**: подпись `username` (или «Аккаунт», если username пуст) → меню с пунктом «Выйти» → вызывает `onLogout` (по решению пользователя, только Logout).
- Реализация меню — MUI `Menu` (якорь к кнопке-триггеру), `MenuItem` с навигацией через `onClick={() => navigate(...)}`. Закрытие меню после выбора.
- Переходы — через `useNavigate` (не `Link`, т.к. внутри `Menu` удобнее программно). Регистрация/Логин/Логаут — те же маршруты (`/register`, `/login`), что и сейчас.
- Меню само не вызывает logout — только `onLogout` от родителя (AppTopBar передаст существующий `useLogout()`), т.е. auth-логика инкапсулирована в существующем хуке.

### 5. `AppTopBar.tsx` (изменить — композитор)
Остаётся каркасом `<AppBar><Toolbar>` и композитором:
- **Слева**: бургер `<MobileNav/>` (mobile only) + бренд «ImiTrade» + иконка About (как сейчас). Бренд+About оборачиваю в `{ display: { xs: 'flex', md: 'flex' } }` — всегда видны.
- **По центру**: `<DesktopNav/>` (md+ only).
- **Справа** (`ml: 'auto'`): блок «Активы» (всегда показывается по решению пользователя, но компактнее на mobile — без подписи «Активы:», только сумма, через адаптивный `sx`) + `<UserMenu onLogout={logout}/>`.
- `useLogout()` вызывается здесь (родитель), не в UserMenu — UserMenu остаётся чисто UI.
- Chip «Пользователь/Гость» убираю — его роль берёт на себя триггер UserMenu (avatar + подпись), дублирование ни к чему.

### Адаптивность — через MUI breakpoints
- Только `sx` media-запросы (`display: { xs, md }`) для показа/скрытия блоков. `useMediaQuery` НЕ нужен — скрытие через CSS-брейкпоинты надёжнее (нетFOUC при ресайзе, нет re-render). Это соответствует ТЗ («использовать MUI breakpoints»).
- Брейкпоинт переключения: `md` (≥900px) — desktop навигация, ниже — бургер (как просит ТЗ: desktop md и выше, mobile xs/sm).

### Что НЕ меняется
- Маршруты (`AppRouter.tsx`), auth-стор (`authStore.tsx`), `useLogout`, `useAccountQuery`, WelcomeDialog, страницы — без изменений.
- Дизайн-токены и цвета — без изменений (синий primary).
- Логика авторизации — переиспользуются `useAuth`, `useLogout`, те же маршруты `/login`, `/register`.

### Проверка
- `npm run build` (tsc + vite) — без ошибок TS.
- `npm run lint` — без новых ошибок/предупреждений в новых/изменённых файлах.
- Desktop (md+): все 4 пункта по центру, About у бренда, Активы + UserMenu справа — как сейчас плюс UserMenu.
- Mobile (xs/sm): бургер слева, бренд+About, Активы (компактно), UserMenu-триггер справа; Drawer открывается, пункты ведут по маршрутам, Drawer закрывается после клика.
- Auth-сценарии: Guest→(Register|Login); Auth→Выйти→Guest (через существующий `useLogout`).
