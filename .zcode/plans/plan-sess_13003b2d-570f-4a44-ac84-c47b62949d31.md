# План: интерактивный зум колесом + вертикальная пунктирная линия + маркер

## Решение конфликта (по ответу пользователя)
Колесо = зум относительно курсора. Wheel-триггер догрузки истории убирается; догрузка старых данных остаётся доступной через **ЛКМ-pan** за левый край (`subscribeVisibleLogicalRangeChange` → `maybeTriggerLazyLoad`, без изменений) и фоновую предзагрузку (без изменений). Требование ТЗ «не изменять существующую логику lazy loading» соблюдено: ни одна функция в `chartZoom.ts` и ни один путь `requestLazyLoad` не меняются — меняется только тот, кто их дёргает.

## Технические решения (обоснование)
- **Зум**: включить встроенный `handleScale.mouseWheel: true` в `chartTheme.ts`. Lightweight Charts из коробки масштабирует логическую шкалу времени **относительно позиции курсора**, плавно и с собственным `preventDefault` — поэтому страница вне графика скроллится как раньше, без отдельного non-passive listener. Это именно поведение из примера ТЗ (точка под курсором остаётся визуальным центром).
- **Вертикальная пунктирная линия + маркер**: собственный overlay (2 DOM-узла, `pointerEvents: 'none'`, `z-index` под tooltip), управляемый из **существующего** `onMove`/`onLeave`. Даёт: чёрную пунктирную линию (`border-left: 1px dashed #000`) во всю высоту холста; круглый маркер в фирменном цвете `#3dba8d` с CSS-анимацией появления (`opacity`+`transform` transition). Позиционирование через уже имеющиеся `chart.timeScale().coordinateToTime(x)` → `timeToCoordinate` (для снапа к точке данных) и `series.priceToCoordinate(value)` — единый путь с tooltip, без новых источников правды. ТЗ это явно разрешает («реализовать собственный overlay без нарушения архитектуры»).
- **Маркер и tooltip** движутся из одного и того же `onMove`, который уже сейчас стреляет каждый пиксель без React re-render (всё в refs/DOM) — лишних перерисовок нет. Маркер и линия скрываются в `onLeave` и при выходе за пределы холста (как tooltip).

## Изменения по файлам

### 1. `frontend/src/features/stock-details/ui/chart/chartTheme.ts`
В `chartBaseOptions.handleScale` (строки 63-68):
- `mouseWheel: false` → **`true`** (включает встроенный зум колесом относительно курсора).
- `handleScroll.mouseWheel` (строка 58) **остаётся `false`** — иначе колесо начинает «скроллить» график по шкале времени; нам нужно именно масштабирование, а не панорамирование колесом. Pan остаётся только через `pressedMouseMove: true`.
- В комментариях блока `handleScale`/`handleScroll` (строки 51-68) обновить пояснение: колесо теперь зумит силами библиотеки, pan — ЛКМ-драг, lazy-load-триггером колеса больше нет.

### 2. `frontend/src/features/stock-details/ui/StockPriceChart.tsx`
**(a) Удалить wheel-effect для lazy-load** (строки 248-266 целиком): его заменяет встроенный зум. `container.addEventListener('wheel', ...)` больше не нужен. Это также убирает non-passive listener и его cleanup.

**(b) Создать overlay-узлы рядом с tooltip** (в effect создания графика, строки 118-125):
- После создания `tooltip` создать `vLine = document.createElement('div')` (`[data-chart-crosshair-vline]`) и `marker = document.createElement('div')` (`[data-chart-crosshair-marker]`), повесить на них inline-стили из новых констант `vLineStyle` / `markerStyle`, `appendChild` в `container`. Хранить в `vLineRef` / `markerRef` (по образцу `tooltipRef`).
- В cleanup (строки 173-187) добавить `vLine.remove()`, `marker.remove()` и обнуление refs — гарантия удаления слушателей/узлов при размонтировании (ТЗ «обработчики корректно удаляются»).

**(c) Расширить `onMove`** (строки 132-157) — после успешного получения `value` (есть точка данных под курсором):
- Позиционировать vLine: `left = series.timeToCoordinate(time)` px (снап к точке данных, чтобы маркер точно лежал на линии графика), `display: 'block'`.
- Позиционировать marker: те же `left` (X точки данных) + `top = series.priceToCoordinate(value)` (Y цены) — центрировать через CSS `transform: translate(-50%,-50%)`.
- В ветках «нет данных / курсор вне холста» (строки 137-151) — скрывать оба узла (`display='none'`) так же, как сейчас скрывается tooltip.
- `onLeave` (строки 158-160) — добавить скрытие vLine и marker.

**(d) Заменить `ensureTooltipStyle`/`TOOLTIP_STYLE_ID`** на единую инъекцию CSS для tooltip + vLine + marker: один `<style id="imitrade-chart-crosshair-style">` с правилами для `[data-chart-tooltip] .cct-label` (существующее) + анимацией появления маркера (`@keyframes` / `.cct-marker { transition: opacity 120ms, transform 120ms }`). Анимация появления — CSS, без JS-таймеров.

**(e) Добавить константы стилей** (рядом с `tooltipStyle`, строки 577+):
- `vLineStyle: Partial<CSSStyleDeclaration>` — `position:absolute`, `top:0`, `bottom:0`, `width:0`, `borderLeft:'1px dashed #000'`, `pointerEvents:'none'`, `display:'none'`, `zIndex:'3'` (под tooltip `4`, над canvas).
- `markerStyle` — `position:absolute`, `width:10`, `height:10`, `borderRadius:'50%'`, `background:'#3dba8d'`, `boxShadow:'0 0 0 3px rgba(61,186,141,0.2)'` (мягкий хало, опц.), `pointerEvents:'none'`, `display:'none'`, `zIndex:'4'`, `transform:'translate(-50%,-50%) scale(0.6)'`, `opacity:0`, transition в инъекционном CSS.
