# Portfolio Value History — Implementation Plan

## Overview
Add a **Portfolio Value** time-series chart to the Portfolio page. Backend reconstructs the historical market value of the user's held positions (`Σ qty_held × close_price` per time point) by replaying transactions and reusing the existing MOEX history integration. Frontend renders it via a **generalized** version of the existing Lightweight Charts component, reusing the same theme, crosshair/tooltip/marker overlay, and period switcher.

No cash balance is included (per decision: pure investment portfolio value).

---

## Backend

### 1. NEW `portfolio/dto/PortfolioHistoryResponse.java`
A `record` mirroring `CandleResponse`'s shape but with `value`:
```java
public record PortfolioHistoryResponse(Instant time, BigDecimal value) { }
```
- `@Schema(name = "PortfolioHistoryResponse")` + Javadoc, matching the DTO conventions (records in `dto/`, never expose entities).
- JSON shape matches the spec: `[{"time":"2026-07-01T00:00:00Z","value":23154.42}, …]`.

### 2. MODIFY `transaction/domain/TransactionRepository.java`
Add one non-paginated accessor for replaying a user's full trade history (currently only `JpaSpecificationExecutor` + paginated `TransactionService.getTransactions` exist):
```java
List<Transaction> findByUserIdOrderByCreatedAtAsc(Long userId);
```
This avoids in-memory filtering and gives the replay a clean chronological stream.

### 3. MODIFY `portfolio/domain/PortfolioService.java`
Add three constructor deps: `TransactionRepository`, `StockHistoryService` (reuse — **no** new MOEX client), `Clock`. Add method:

```java
@Transactional(readOnly = true)
public List<PortfolioHistoryResponse> getHistory(Long userId, HistoryPeriod period, LocalDate from)
```

Algorithm (per spec §"Формирование данных", with local per-ticker cache to avoid N×M):
1. Load all user transactions (`findByUserIdOrderByCreatedAtAsc`). If empty → return `List.of()` (empty portfolio → frontend shows Empty State).
2. Resolve distinct `stockId`s → batch `stockRepository.findAllById(...)` → `Map<stockId, ticker>`.
3. **Local cache** `Map<String, List<CandleResponse>>` keyed by ticker. For each distinct ticker, call `stockHistoryService.getHistory(ticker, period, from)` **once** and store.
4. Build the union of all candle timestamps → sorted ascending `times[]`.
5. For each stock, precompute a prefix-sum quantity timeline from its transactions (`qtyAt(stockId, t) = Σ(+qty BUY / −qty SELL) for createdAt <= t`).
6. For each `t` in `times[]`:
   - `value(t) = Σ_stock qtyAt(stock, t) × closeAt(stock, t)` where `closeAt` = last candle `close` with `time <= t` (forward-fill via `headMap(t, true).lastEntry()`; skip stock if no candle yet).
   - Skip the point if total quantity held at `t` is 0 (before first buy / after full sell-out) — avoids a long flat zero line.
   - Accumulate raw, then `setScale(MONEY_SCALE=4, HALF_UP)` once at the end (mirrors `AccountService`).
7. Return `List<PortfolioHistoryResponse>` ascending by time.

The math mirrors `AccountService` (§"portfolioValue = Σ(quantity × currentPrice)") but with historical closes and replayed quantities. No persistence, no schema changes (consistent with project philosophy). MOEX failures propagate as 503 via existing `GlobalExceptionHandler` (`MarketDataUnavailableException`) — **no handler/error-code changes**.

### 4. MODIFY `portfolio/web/PortfolioController.java`
Add sibling endpoint mirroring `StockHistoryController` param style:
```java
@GetMapping("/history")
public ResponseEntity<List<PortfolioHistoryResponse>> getHistory(
        @AuthenticationPrincipal AuthenticatedUser principal,
        @RequestParam(name="period", required=false, defaultValue="1D") String period,
        @RequestParam(name="from", required=false) LocalDate from) {
    HistoryPeriod parsed = HistoryPeriod.parse(period);
    return ResponseEntity.ok(portfolioService.getHistory(principal.userId(), parsed, from));
}
```
Full OpenAPI annotations (`@Operation`, `@ApiResponses` with 200/401) matching the existing method. No security-config change — `/api/v1/portfolio/**` is already covered by the default "authenticated" rule (JWT or X-Guest-Token).

### 5. NEW backend test `PortfolioHistoryServiceTest.java`
Unit test (`@ExtendWith(MockitoExtension.class)`), following the **explicit-constructor + fixed `Clock`** pattern from `StockHistoryServiceTest` (since `@InjectMocks` can't inject a fixed clock). Mocks `TransactionRepository`, `StockHistoryService`, `StockRepository`, `PortfolioPositionRepository`. Covers every case from spec §"Тесты":
- empty portfolio → `[]`
- one stock, single buy
- several stocks
- multiple buys of the same stock (quantity accumulates)
- partial sell (quantity decrements, line continues)
- full sell (quantity hits 0 → subsequent points skipped)
- math correctness (assert `value` equals expected `Σ qty×close` with scale 4)

### 6. MODIFY existing `PortfolioServiceTest.java`
Add `@Mock Clock clock` field so `@InjectMocks` still wires up after the new constructor dep (existing `getPortfolio` tests are unaffected — they don't touch `Clock`). Also extend `PortfolioControllerTest.java` (web slice) with a `GET /history` test using the existing `JwtAuthentication` + `SecurityContextHolder` install pattern, stubbing `portfolioService.getHistory(...)` and asserting `jsonPath` on the response shape.

## Frontend

### Architecture decision: generalize via `shared/`
The existing `StockPriceChart.tsx` is hardcoded to `getStockHistory(ticker, …)` and lives inside `features/stock-details/`. Per the frontend CLAUDE.md **feature-isolation rule** ("features never import each other's internals; cross-feature communication only via `shared/`"), the generalized chart primitive and its helpers move to `shared/`. Both `StockPriceChart` and the new `PortfolioValueChart` become thin wrappers.

### 7. MOVE `features/stock-details/ui/chart/chartTheme.ts` → `shared/lib/chart/chartTheme.ts`
Pure, ticker-agnostic (exports `CHART_LINE_COLOR`, `AXIS_TEXT_COLOR`, `chartBaseOptions`, `areaSeriesOptions`). Update the import in `StockPriceChart`.

### 8. MOVE `features/stock-details/ui/chart/chartZoom.ts` → `shared/lib/chart/chartZoom.ts`
Pure date/viewport utilities (`PERIOD_LOOKBACK_DAYS`, `CANONICAL_PERIODS`, `createInitialViewport`, `addDays`, `toTimestamp`, `needsLazyLoad`, `nextLazyLoadFrom`, `throttle`, …). Update imports.

### 9. NEW `shared/components/charts/PriceLineChart.tsx` — the generalized base
Extracted from `StockPriceChart`. Props:
```ts
interface PriceLineChartProps {
  seriesKey: string;                       // remount key (ticker | 'portfolio')
  fetchData: (period, from?) => Promise<HistoryPoint[]>;   // pluggable source
  queryKey: (period, fromIso?) => readonly unknown[];       // React Query key
  lazyLoad?: boolean;                       // default true (stock), false (portfolio)
  tooltipValueLabel?: string;               // default 'Price', portfolio passes 'Стоимость'
  period?: HistoryPeriodCode;               // initial period, default '1D'
}
```
Preserves **exactly**: chart creation + `chartBaseOptions`/`areaSeriesOptions`, period switcher buttons (`#3dba8d` active), crosshair DOM-overlay (black dashed vLine + brand marker + tooltip), min/max/current Y-axis price lines, loading/error/empty overlays, `autoSize`. Lazy-load machinery (`maybeTriggerLazyLoad`, `requestLazyLoad`, background preload) runs only when `lazyLoad === true`; otherwise the chart just plots the single fetched range. The tooltip's value line uses `formatMoney` (works for both price and portfolio value) and the customizable label.

### 10. MODIFY `features/stock-details/ui/StockPriceChart.tsx`
Becomes a ~20-line wrapper:
```tsx
export default function StockPriceChart({ ticker }: { ticker: string }) {
  return (
    <PriceLineChart
      seriesKey={ticker}
      fetchData={(period, from) => getStockHistory(ticker, from, period)}
      queryKey={(period, fromIso) => queryKeys.stockHistory(ticker, period, fromIso)}
      lazyLoad
      tooltipValueLabel="Цена"
    />
  );
}
```
Behavior identical to today (verified by `npm run build` + manual check that the stock chart still zooms/lazy-loads).

### 11. MODIFY `shared/lib/queryKeys.ts`
Add:
```ts
portfolioHistory: (period: string) => ['portfolio', 'history', period] as const,
```
Note: `queryKeys.portfolio = ['portfolio']` is the prefix — React Query prefix invalidation in `useBuyStockMutation`/`useSellStockMutation` already invalidates `['portfolio']`, which **automatically captures** `['portfolio','history',…]`. So after a trade the history refreshes for free — **no mutation changes needed**.

### 12. MODIFY `shared/api/endpoints.ts`
Add `PORTFOLIO.history: '/api/v1/portfolio/history'`.

### 13. NEW `features/portfolio/types/portfolioHistoryTypes.ts`
```ts
export interface PortfolioHistoryPoint { time: string; value: number; }
```

### 14. NEW `features/portfolio/api/portfolioHistoryApi.ts`
`getPortfolioHistory(period): Promise<PortfolioHistoryPoint[]>` — calls `apiClient.get(API_ENDPOINTS.PORTFOLIO.history, { params: { period } })`, maps `{ time, value: Number(...) }`. (No `from` sent — period switcher only, no lazy load.)

### 15. NEW `features/portfolio/model/usePortfolioHistoryQuery.ts`
`usePortfolioHistoryQuery(period)` — `useQuery({ queryKey: queryKeys.portfolioHistory(period), queryFn: () => getPortfolioHistory(period) })`. Default QueryClient options apply.

- When portfolio is **non-empty**: render the chart in a `<Card>`/`<CardContent>` (AccountSummary `StatCard` pattern) **above** `<PortfolioTable>`, titled «Стоимость портфеля». Header + chart + table on the existing `<Container maxWidth="lg">`.
