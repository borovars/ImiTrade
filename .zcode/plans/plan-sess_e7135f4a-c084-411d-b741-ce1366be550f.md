## План изменений

### Часть 1 — Логотип во весь размер (без обрезки)
Единственное место, где рендерится логотип — `frontend/src/features/stock-details/ui/StockDetailsView.tsx` (Avatar 64×64). SVG обрезается из-за дефолтного `object-fit: cover` у MUI Avatar.

**`frontend/src/features/stock-details/ui/StockDetailsView.tsx`**
- Добавить `objectFit: 'contain'` в `sx` `<Avatar>` (строка 71). Один проп, fallback на `default.svg` сохраняется. `bgcolor: 'background.default'` остаётся для аккуратных полей SVG.

(В `StocksTable`/`PortfolioTable` логотипов нет — там ничего не трогаем.)

### Часть 2 — Сортировка на /stocks (backend-side, Select с фиксированными направлениями)
Backend уже принимает `sort` через `@PageableDefault Pageable` (`StockController.getStocks`) → `findAll(spec, pageable)`. Валидные поля: `ticker`, `currentPrice`, `lotSize`. **Изменений backend нет.** Сортировка только серверная — каталог пагинируется по 20 строк, клиентская сортировка переупорядочила бы только текущую страницу.

Три варианта Select с фиксированным направлением:
| Лейбл | sort |
|---|---|
| Default (A→Z) | `ticker,asc` (новый стабильный дефолт) |
| Price | `currentPrice,desc` (дорогие→дешёвые) |
| Lot Size | `lotSize,asc` (меньше→больше) |

**`features/stocks/api/stocksApi.ts`** — `getStocks(page, size, sort = 'ticker,asc')`: добавить `sort` в `params: { page, size, sort }`.

**`features/stocks/model/useStocksQuery.ts`** — `useStocksQuery(page, size, sort)`: вложить `sort` в query-key `[..., { page, size, sort }]` (отдельный кэш на сортировку, `keepPreviousData` сохраняется); прокинуть в `getStocks`.

**`pages/StocksPage.tsx`**:
- `const [sort, setSort] = useState('ticker,asc')` (дефолт — алфавит).
- Передать `sort` в `useStocksQuery`; `handleSortChange` сбрасывает `page` в 0.
- MUI `Select` «Sort by» с тремя опциями над таблицей, заблокирован при `isPlaceholderData`.

### Поведение
- Логотип на Stock Detail показывается целиком (contain).
- `/stocks` стабильно грузится по алфавиту (Default, A→Z); Price → дорогие→дешёвые; Lot Size → меньше→больше; смена сортировки сбрасывает на 1-ю страницу; пагинация корректна.
- Без новых файлов, без правок типов/backend/диалогов/портфеля.
