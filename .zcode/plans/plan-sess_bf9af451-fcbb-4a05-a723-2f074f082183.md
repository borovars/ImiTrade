## План: правки визуала графика акции

### Причина бага «шрифт не увеличивается»
В `chartTheme.ts` стоит `fontFamily: 'inherit'`. Библиотека `lightweight-charts` собирает canvas-строку шрифта как `` `${size}px ${family}` `` → `"24px inherit"`. Canvas 2D API не понимает ключевое слово `inherit` в свойстве `font`, считает строку невалидной и молча игнорирует, откатываясь к дефолту canvas (~12px). Поэтому `AXIS_FONT_SIZE` хоть и = 24, реально рендерится мелко.

### Изменения

**Файл 1: `frontend/src/features/stock-details/ui/chart/chartTheme.ts`**
1. **Фикс шрифта (главное)** — заменить `fontFamily: 'inherit'` на реальное системное семейство (библиотечный дефолт), чтобы `makeFont` производил валидную canvas-строку и `fontSize` применялся.
2. **Увеличить шрифт осей** — поднять `AXIS_FONT_SIZE` с 24 до **32** (пользователь выбрал крупный, но 32 — безопасный вариант без поломки вёрстки на узких экранах).
3. **Убрать пунктирный крест + подписи на осях при наведении** — обнулить секцию `crosshair`:
   - `vertLine.visible = false`, `vertLine.labelVisible = false`
   - `horzLine.visible = false`, `horzLine.labelVisible = false`
   (Сам crosshair-движок оставляем включённым — он нужен для отлова позиции курсора в tooltip.)
4. **Линию жирнее** — `lineWidth: 2 → 3` в `areaSeriesOptions` (выбрано пользователем).

**Файл 2: `frontend/src/features/stock-details/ui/StockPriceChart.tsx`**
1. **Плавающее окно tooltip** — добавить подписку `chart.subscribeCrosshairMove(handler)`:
   - рендерим отдельный абсолютно-позиционированный `<div>` (создаём/кэшируем через `ref`, как рекомендует readme: «пишется в DOM напрямую, без re-render»), следящий за курсором внутри контейнера;
   - внутри окна: дата (через существующий `formatDateTime` из `@/shared/utils/format`) и цена (`formatMoney`) точки, на которую наведён курсор;
   - позиция окна считается из `param.point` (canvas-координаты) со смещением, чтобы не уходить за правый/нижний край контейнера;
   - данные берутся из `param.seriesData` (или из `mergedRef` по `param.time`) — без повторной выборки сети;
   - при уходе курсора с графика (`param.point === undefined`) окно скрывается;
   - отписка в cleanup-функции эффекта создания графика (`unsubscribeCrosshairMove`).

### Что НЕ трогаю
- Логику lazy-load, price lines (min/max/current на оси Y), кнопки периода, состояния loading/error/empty, зум/панорамирование.
- Подписи на оси Y остаются (min/max/current) — их генерируют price lines, а не crosshair; пользователь просил убрать только crosshair-подписи при наведении.

### Проверка
- `npm run lint` и `npm run build` (tsc strict) во frontend.
- Визуальная проверка: шрифт осей реально крупный, линия толще, при наведении — плавающее окно с датой+ценой, пунктирного креста и подписей на осях нет.
